# Case Study Template

This is the **portfolio-ready** template used to generate and edit case studies.

> Format: Markdown first. Export to PDF later.

## Title
- `{Scenario Name} — {Key Problem} (Simulation Case Study)`

## 1. Context
- Company/product background (from scenario context)
- Market / users
- What triggered this initiative

**Auto-fill sources:** Scenario context + run step 1 prompt

## 2. Problem Statement
- What is the core user/business problem?
- How do we know it’s a problem (signals, data, qualitative insights)?

**Auto-fill sources:** early events + baseline metrics

## 3. Constraints & Goals
- Constraints: timeline, team, compliance, tech limitations, budget
- Goals/KPIs and success criteria

**Auto-fill sources:** scenario config + run state

## 4. Options Considered
For each option:
- summary
- pros/cons
- risks
- expected KPI impact

**Auto-fill sources:** decision history (choices) + rationale + risk lists

## 5. Decision & Rationale
- What you chose and why
- How you aligned stakeholders
- What you explicitly *did not* do

**Auto-fill sources:** key decision steps + stakeholder events

## 6. Execution Plan
- milestones
- experiment plan (if applicable)
- rollout strategy and monitoring

**Auto-fill sources:** planning/execution steps

## 7. Results
- KPI movement over time (before/after)
- notable events (incidents, stakeholder shifts)
- what surprised you

**Auto-fill sources:** metric snapshots + events timeline

## 8. Reflection
- what you would do differently
- lessons learned tied to learning objectives

**Auto-fill sources:** debrief output + user-authored text

---

## Appendix (optional)
- timeline table
- metrics chart screenshots
- links to artifacts created during the run
